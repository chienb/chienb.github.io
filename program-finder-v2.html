Program Finder v2

CSUDH

2019

Intuitive sorting of degree programs for students all on a single page

The Challenge

V1 allowed students to see and filter programs based on different categories like degree, area of study, college, etc. I tried to include as many programs as I can into a 4-column grid system color-coded to help students identify programs that belong to a specific category filter. This proved to be overwhelming for our student pilot groups. So, back to the drawing board…

Ideation

During my design process, I often times like to approach a problem using the CFTBS approach. The idea is to complete the sentence Create ___ for ___ to ___ by ___ so ___. This helps me to focus on problems I want to solve and how to approach it. After brainstorm a few keywords with the team, here's what we ended up with for this project: CREATE a simple yet intuitive way FOR students TO find information on degree programs BY creating a sortable list of programs SO that they can find all relevant information on a single page.

User Flow - minimizing goal steps

V1 was great in allowing students to see a list of all programs but viewing program description required an additional step. The only way to navigate to the description is by clicking on “explore options” which will display a modal with more information. We want to avoid this because every step in a goal funnel is a potential bounce risk

"The information that I’m seeking is not available to me immediately, I don’t want to bother with clicking more links"

Paper Wireframes

Ultimately, what we are trying to accomplish here is to provide as much program detail as we can and make the list sortable and filterable. In order to accomplish this, I started by listing out all the important information that a student would be looking for. With the help of the team, we picked out the most important information that are unique to each program.

Mockups

I used a libary called Isotope which was originally created for galleries and imagery for its sorting and filtering functions. The CSS transitions are also smooth and fast. Students are now able to view more info on the same page and are more likely to perform an action (clicking on CTA).
